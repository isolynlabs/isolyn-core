# Security Policy

Isolyn Core is designed with security and separation as first-class
architectural concerns.

We take security issues seriously and encourage responsible disclosure.

---

## Reporting a Vulnerability

If you discover a potential security vulnerability, please do not open
a public issue.

Instead, report it privately to:

security@isolyn.fun

Please include:
- a clear description of the issue
- steps to reproduce, if applicable
- potential impact assessment

We will acknowledge receipt and investigate promptly.

---

## Scope

This security policy applies to:
- core infrastructure components
- execution and privacy interfaces
- reference implementations published in this repository

Third-party dependencies and external services are governed by their
respective security policies.

---

## Disclosure

We request that reporters allow reasonable time for investigation and
remediation before any public disclosure.
